# 🟡 M3-触觉表征与力预测

> **核心问题：** 触觉图像怎么编码？力/力矩信号怎么预测？不同触觉传感器的数据能互通吗？仿真里训练的力觉模型能迁移到真机吗？
>
> **预计时间：** 2 周
>
> **状态：** 📝 待学习

---

## 1. 触觉不是"另一种视觉"

```
视觉：      高维像素 → 物体在哪、长什么样
触觉：      接触形变 → 接触力多大、滑不滑、什么材质

关键区别：
  → 视觉是非接触的（光照变化、遮挡...）
  → 触觉只在接触时有信号（但物理规律不变——泛化更容易！）
  → "位置预测大家在卷，力控才是蓝海"
```

## 2. 触觉表征学习

### 触觉图像 → 紧凑表征

```
GelSight/DIGIT (160×120×3) → CNN/ViT Encoder → 128-D embedding

预训练方法：
  1. MAE：随机 mask 触觉图像 patches → 重建（自监督，不需要标签）
  2. 对比：同一物体视觉 embedding · 触觉 embedding → 正对（跨模态对齐）
  3. 力预测：触觉 image → 力/力矩 → 监督学习（需要 F/T 传感器标签）
```

### 跨传感器迁移（GenForce, Nature Comms 2026）

```
在 GelSight 上训练的力预测模型
  → 迁移到 DIGIT、TacTip、uSkin
  → 法向力误差降低 80%

→ 你的多传感器方案：不同硬件用不同触觉传感器的数据可以互通
→ 预训练触觉表征 + 跨硬件迁移是可行的
```

## 3. 力/力矩的表示与预测

### 力信号的预处理

```
6-DoF wrench: F = [f_x, f_y, f_z, τ_x, τ_y, τ_z] @ 1000Hz

预处理：
  → 重力补偿：减去末端工具自重（需要知道质量+质心）
  → 低通滤波：去掉高频机械振动
  → 外部力提取：F_ext = F_raw - F_gravity - F_inertia

输入网络时：降采样到统一频率（如 100Hz）
```

### 力预测训练

```
输入：触觉图像 + 关节状态 + 当前力
输出：未来 k 步的力/力矩

自监督预训练（参考 3D Tactile ICRA 2025）：
  → 随机 mask 力序列的时间窗口 → 重建
  → encoder 学会力动力学

Touch Dreaming (HTD, CMU 2025)：
  → 在 latent 空间预测触觉/力
  → 不需要大量真实触觉数据
```

## 4. Sim-to-Real 力触觉

### 为什么力触觉的 Sim-to-Real 比视觉难？

```
视觉 Sim-to-Real：渲染引擎能做到以假乱真（domain randomization）
力触觉 Sim-to-Real：接触动力学极其敏感
  → 摩擦系数（0.3 vs 0.35 → 抓取结果完全不同）
  → 接触刚度（橡胶的硬度仿真很难调）
  → 传感器噪声模型（每个传感器不一样）
```

### 解决方案（Zero-Shot Dexterous Force, 2026.01）

```
1. 快速触觉仿真：密集虚拟触觉单元
2. 电流→力矩标定：省去力矩传感器
3. Domain Randomization：摩擦、刚度、传感器噪声全部随机
4. 结果：完全仿真训练 → 零样本真机部署
```

## 5. 自适应视觉-触觉融合

### 力引导的注意力（Adaptive Visuo-Tactile Fusion, 2025.05）

```
不同操作阶段 → 不同模态权重：

预接触阶段：视觉主导（"物体在哪"），触觉无信号 → 路由到 Vision Expert
接触瞬间：力信号突变 → Router 迅速切换到 Force + Tactile Expert
操作中：  力触觉主导（"抓稳了吗"）

→ 这就是你的 MoT Router 应该学到的行为
→ 不需要手动设权重，网络自己学会
→ 真实数据验证：3 个任务 93% 成功率
```

## 6. 力触觉进网络结构：六种设计范式

> **这是面试高频问题："力触觉信号怎么进你的网络？"** 本节梳理现有好工作的六种做法，并给出比较分析。

### 6.1 范式总览

```
输入层融合              特征层融合              决策层融合
(Early Fusion)         (Late Fusion)          (Late Fusion v2)
                          
力 ─┐                   力→[MLP]─┐             力→[MLP]→ 力特征 ─┐
触 ─┼→ concat → 网络    触→[CNN]─┼→ concat →   触→[CNN]→ 触特征 ─┼→ 策略
视 ─┘                   视→[ViT]─┘  共享骨干    视→[ViT]→ 视特征 ─┘
                          
最粗暴                  最常用                   最高级
                          
                          
Token化融合             传感器掩码融合           自适应/预测融合
(Token-based)          (Sensor-Masked)         (Adaptive/Predictive)

力→[离散化]→token      力→渲染到图像通道         力预测 → 引导注意力权重
触→[patch化]→token     触→叠加到RGB像素          视+触 → 力预测 → Router
视→[patch化]→token     视→RGB图像                各模态权重动态调整
   ↓                       ↓                        ↓
统一Transformer        冻结的ViT编码器            MoE路由/Cross-Attn
```

### 6.2 逐范式详解

#### 范式 1：输入层融合 (Early Fusion / Concatenation)

```
做法：把所有模态 concat 成一个向量 → 进一个网络

示例：
  state = [joint_pos, joint_vel, ft_wrench, tactile_flat, image_flat]
  action = MLP(state)

代表工作：
  - 大多数 BC baseline
  - 早期 RL for manipulation

优点：
  ✅ 极简，实现成本为零
  ✅ 所有模态共享梯度

缺点：
  ❌ 维度灾难——6D力信号被10^5 D视觉淹没
  ❌ 频率不同（力1000Hz vs 视觉30Hz）→ 必须粗暴降采样
  ❌ 缺失模态无法优雅处理
  ❌ 没有模态间的结构化交互

什么时候用：
  → 快速原型验证"力信号有没有用"
  → 模态维度相近时（如只有本体感觉+力，没有视觉）
```

#### 范式 2：特征层融合 (Late Fusion / Separate Encoders)

```
做法：每种模态独立的 encoder → concat 特征 → 共享 backbone

示例：
  force_embed  = ForceMLP(ft_wrench)          # (B, 64)
  tactile_embed = TactileCNN(tactile_image)    # (B, 128)
  visual_embed  = ViT(rgb_image)              # (B, 512)
  
  fused = concat([force_embed, tactile_embed, visual_embed])
  action = Transformer(fused + pos_embed)

代表工作：
  - Diffusion Policy with multi-modal input (Chi et al. 2023 的扩展做法)
  - 大多数多模态 robot policy 的默认架构

优点：
  ✅ 每种模态独立的编码器——可以针对信号特性设计（力用MLP、触觉用CNN、视觉用ViT）
  ✅ 频率天然解耦（各自在自己频率下编码，concat 时对齐）
  ✅ 可以单独预训练各 encoder（如触觉用 MAE 预训练）

缺点：
  ❌ concat 后模态间无结构化交互——只有 backbone 的 self-attention 在学习关系
  ❌ 缺失模态时需要零填充或特殊处理
  ❌ 力信号在大量视觉 token 中仍然可能被稀释

什么时候用：
  → 模态种类 > 3 且维度差异大
  → 这是你当前框架的 baseline 设计
```

#### 范式 3：Token 化融合 (Token-based / Unified Transformer)

```
做法：所有模态统一 token 化 → 进入同一个 Transformer

示例（RT-2 式离散化）：
  action_tokens = discretize(action, bins=256)
  force_tokens  = discretize(ft_wrench, bins=256)
  image_tokens  = ViT.patch_embed(rgb_image)
  
  all_tokens = [image_tokens, force_tokens, action_tokens]
  output = GPT(all_tokens)

示例（π₀ 式连续 FM）：
  force_token   = MLP_proj(ft_wrench)          # → (B, 1, d_model)
  tactile_tokens = ViT(tactile_image)           # → (B, N_tac, d_model)
  visual_tokens  = ViT(rgb_image)              # → (B, N_vis, d_model)
  
  all_tokens = concat([visual_tokens, force_token, tactile_tokens])
  output = DiT(all_tokens)

代表工作：
  - RT-2 (discrete)
  - π₀ (continuous, flow matching head)
  - OpenVLA (discrete)
  - Octo (continuous, diffusion head)

优点：
  ✅ 统一的 token 空间——self-attention 自然处理跨模态交互
  ✅ 可以无缝接入预训练的 VLM backbone（π₀、RT-2 的做法）
  ✅ 缺失模态 = 去掉对应的 token（优雅）
  ✅ 适合 MoE——不同模态 token 走不同 expert

缺点：
  ❌ 力信号 token（1-2 个 token）vs 视觉 token（数百个）→ 数量失衡
  ❌ 离散化丢精度（RT-2 式）
  ❌ 长序列 → Transformer O(N²) 代价

什么时候用：
  → 你要用预训练 VLM 做 backbone 时
  → 模态种类多、需要有结构化的跨模态交互时
  → 这是你的 MoT 架构的底座
```

#### 范式 4：传感器掩码融合 (Sensor-Masked Fusion)

```
做法：把力/触觉数据渲染成图像通道，叠加到 RGB 图像上 → 冻结的 ViT 编码

示例（OmniVLA, 2025）：
  # 力的 6D → 渲染为 2 通道图像（大小=视觉特征图）
  force_map = render_to_image(ft_wrench, size=vis_feature_size)
  
  # 触觉 → 直接是图像，resize 到统一尺寸
  tactile_resized = resize(tactile_image, size=vis_feature_size)
  
  # 通道拼接
  augmented_image = concat_channels([rgb, depth, force_map, tactile_resized])
  
  # 冻结的 ViT（不需要重新训练！）
  features = FrozenViT(augmented_image)

代表工作：
  - OmniVLA (2025): 25% → 84% 成功率提升
  - 2× 样本效率，+59% OOD 泛化

优点：
  ✅ 可以利用冻结的大规模预训练视觉模型（不破坏预训练权重）
  ✅ 数据效率极高（不需要重新学视觉表征）
  ✅ 空间对齐——力和触觉在图像空间中有了物理位置

缺点：
  ❌ 需要空间标定（力传感器在手腕→映射到图像哪个位置？）
  ❌ 力信号不是空间分布的→渲染成图像通道有信息损失
  ❌ 触觉图像和视觉图像的 domain gap——冻结的 ViT 可能不理解触觉纹理

什么时候用：
  → 视觉 encoder 很强但你不想微调
  → 数据量少（<100 条轨迹）
```

#### 范式 5：自适应/预测融合 (Adaptive/Predictive Fusion)

```
做法：用力预测来引导注意力——网络学会"什么时候该信视觉，什么时候该信触觉"

示例（Adaptive Visuo-Tactile Fusion, 2025.05）：
  # 力预测模块
  pred_force = ForcePredictor(visual_embed, tactile_embed)
  
  # 力引导的注意力
  force_attn = cross_attn(
      Q = visual_embed,
      K = tactile_embed,
      attn_bias = pred_force  # ← 力预测结果引导注意力权重
  )
  
  # 阶段自适应
  if pred_force.norm < contact_threshold:
      weight = 0.9 * visual + 0.1 * tactile   # 预接触：信视觉
  else:
      weight = 0.2 * visual + 0.8 * tactile   # 接触中：信触觉

代表工作：
  - Adaptive Visuo-Tactile Fusion (2025): 93% 成功率
  - HTD "Touch Dreaming" (CMU 2025): latent 空间预测触觉，+90.9%
  - 3D Tactile + Force Pretraining (ICRA 2025): 78%

优点：
  ✅ 显式建模"接触状态转换"——正是操作中最关键的瞬间
  ✅ 不需要手动设计模态权重
  ✅ 力预测作为辅助任务 → 额外的训练信号

缺点：
  ❌ 需要力预测模块 → 额外的网络组件和训练目标
  ❌ 力预测本身的精度影响融合质量
  ❌ 计算开销大于范式 2

什么时候用：
  → 你的核心场景（接触操作力控）
  → 这最接近你的 MoT Router 应该学到的行为
```

#### 范式 6：世界模型融合 (World Model / Latent Prediction)

```
做法：学一个"预测未来"的世界模型 → latent state 里自然包含力触觉信息

示例（ABot-PhysWorld 2026, HTD 2025）：
  # 编码
  z_t = Encoder(obs_t)  # obs_t 包含视觉+力+触觉
  
  # 世界模型：预测下一时刻的 latent
  ẑ_{t+1} = WorldModel(z_t, action_t)
  
  # 策略在 latent 空间里操作
  action_t = Policy(z_t, ẑ_{t+1})

代表工作：
  - ABot-PhysWorld (2026): 14B DiT, DPO 后训练抑制非物理行为
  - HTD (CMU 2025): latent touch prediction, +90.9%

优点：
  ✅ latent state 包含丰富的物理信息
  ✅ 可以做"mental rehearsal"——在 latent 空间模拟接触结果
  ✅ 不需要真实力触觉标签（自监督预测）

缺点：
  ❌ 学世界模型本身就是 hard problem
  ❌ 大规模（14B DiT）
  ❌ 在力触觉精度要求高的场景，latent 预测的误差可能不可接受

什么时候用：
  → 数据量大、算力多时
  → 需要模型有"物理直觉"的预训练阶段
```

### 6.3 六种范式对比

| | 实现复杂度 | 力触觉作用 | 跨模态交互 | 缺失模态鲁棒 | 适合你的场景 |
|---|---|---|---|---|---|
| ① 输入层融合 | ★ | 弱（被稀释） | 无 | 差 | 快速原型 |
| ② 特征层融合 | ★★ | 中 | 弱（隐含在 attn） | 中 | **baseline** |
| ③ Token 化融合 | ★★★ | 中 | 强（统一 attn） | 好 | **MoT 底座** |
| ④ 传感器掩码 | ★ | 中 | 中（空间对齐） | 中 | 少数据 |
| ⑤ 自适应融合 | ★★★★ | **强（显式引导）** | **强（力控 attn）** | 好 | **核心方案** |
| ⑥ 世界模型 | ★★★★★ | 强（隐式） | 强（预测驱动） | 好 | 大算力预训练 |

### 6.4 面试怎么讲

> **"力触觉信号怎么进你的网络结构？"**

```
推荐回答框架（3 分钟）：

1. 首先明确问题特殊性（30秒）：
   "力触觉和视觉有三个核心矛盾——频率不同（1000Hz vs 30Hz）、
   维度差距大（6D vs 10^5 D）、接触才有信号。"

2. 分类现有做法（60秒）：
   "现有工作分三种思路——
   特征层融合（最简单，各模态独立 encoder 后 concat）；
   Token 化融合（最主流，π₀/RT-2/OpenVLA 都这么做，
   把所有模态统一 token 化进 Transformer）；
   自适应融合（最新，用力预测引导注意力权重，在接触瞬间
   自动从'信视觉'切换到'信触觉'——OmniVLA 从 25% 到 84%）。"

3. 给出你的判断和选择（60秒）：
   "我的设计是 Token 化 + 自适应融合的结合——
   所有模态 token 化进入 MoE Transformer（范式③），
   但同时加一个轻量的力预测模块（范式⑤），
   预测结果作为 Router 的额外输入，让 MoE 在接触瞬间
   自动把更多 token 路由到 Force Expert 和 Tactile Expert。
   
   这样做的好处是：①统一 token 空间的跨模态交互能力；
   ②自适应权重不需要手工调；③力预测模块很轻量（几个 MLP 层），
   不会显著增加推理延迟。"

4. 补充分析（30秒）：
   "这个设计的主要风险是力预测精度——如果力预测不准，
   自适应权重反而会误导 Router。我的 mitigiation 是
   在预训练阶段用力预测作为辅助 loss，用仿真数据先训好
   力预测模块，真机微调时冻结它。"
```
