## 一、具身基础模型 (Embodied Foundation Models)

### 1.1 整体格局

2025-2026 年，VLA (Vision-Language-Action) 成为主导范式。中美实验室激烈竞争，关键趋势：

| 趋势 | 含义 |
|------|------|
| **跨具身统一** | "一个大脑，多种身体"——同一模型控制单臂、双臂、移动底盘、人形机器人 |
| **3D 空间理解** | 从 2D VLM 升级到 3D-aware VLM，注入空间推理能力 |
| **物理对齐** | 世界模型不仅要"看起来真"，还要"符合物理定律"（不穿透、不反重力） |
| **数据效率** | 用更少的机器人数据达到 SOTA（SPEAR-1 用 20× 更少数据匹敌 π₀） |
| **开源运动** | Spirit v1.5、MiMo-Embodied、SPEAR-1、ABot-M0 全部开源 |

### 1.2 关键模型

#### Spirit AI v1.5（中国杭州，2026.01）

```
里程碑：RoboChallenge 排行榜第一名（首个超 50% 成功率）
  - 总分 66.09，任务成功率 50.33%
  - 超越 Physical Intelligence 的 π₀.5
  - 统一 VLA 架构，端到端感知-推理-动作
  - 开源
  - 配套人形机器人 Moz1

对你的意义：
  → VLA 是你要追赶的范式
  → RoboChallenge 30 个日常任务——你的评估基准参考
```

#### ABot-M0（AMAP CV Lab，2026.02）

```
核心创新：Action Manifold Learning (AML)
  → 假说：有效机器人动作位于低维光滑流形上，而非散落在高维空间
  → 用 DiT (Diffusion Transformer) backbone 直接预测干净动作序列
  → 600万+ 轨迹、9500+ 小时、20+ 具身的大统一数据集 (UniACT)

关键数字：
  - LIBERO: 98.6%
  - LIBERO-Plus: 80.5%
  - RoboCasa: 58.3%

对你的意义：
  → 动作流形假说——你的 SE(3) 运动流形思路和这个高度一致！
  → AML 用的是 Diffusion，你换成 FM 就是自然延伸
  → UniACT 数据集的跨具身思路可以用在你的数据融合设计中
```

#### SPEAR-1（CVPR 2026）

```
核心创新：3D-aware VLM
  → 从单张 2D 图像推断物体 3D 坐标
  → 用 ~45M 帧训练（24 个 Open X-Embodiment 数据集）
  → 匹敌/超越 π₀-FAST 和 π₀.5，但用了 20× 更少的机器人演示数据

对你的意义：
  → 3D 空间理解 → 你的 SE(3) 位姿预测的视觉 backbone 设计参考
  → 数据效率的标杆——"少数据"是你的真机数据约束
```

#### ABot-PhysWorld（2026.03）

```
核心创新：14B DiT 世界模型 + 物理对齐
  → 300 万操作片段训练
  → DPO-based 后训练抑制非物理行为（物体穿透、反重力）
  → 超越 Veo 3.1 和 Sora v2 Pro（物理合理性指标）

对你的意义：
  → 力触觉的世界模型——如果世界模型能预测接触力，就能做力控的 mental rehearsal
  → DPO 后训练——用于你的力控风格微调
```

#### MiMo-Embodied（2025.11）

```
核心创新：首个跨自动驾驶 + 具身智能的统一模型
  → 17 个具身 benchmark SOTA
  → 12 个自动驾驶 benchmark SOTA
  → 正迁移：驾驶数据帮助操作，反之亦然
  → 多阶段学习 + CoT/RL 微调
  → 完全开源

对你的意义：
  → 跨领域预训练的证据——驾驶数据能帮操作
  → CoT (Chain-of-Thought) 微调——可以用于你的操作策略推理
```

## 二、Flow Matching 策略（2025-2026）

### 核心趋势

| 趋势 | 代表工作 |
|------|----------|
| **蒸馏到少步/单步** | ReinFlow、Flow to One Step、FlowPolicy、FlowRAM |
| **在线微调+交互修正** | ReinFlow (RL微调)、FlowCorrect (人在回路) |
| **多模态+3D 感知融合** | FlowRAM、ManiFlow、"Flow to One Step" |
| **超越准静态任务** | Generative Predictive Control、Streaming Flow Policy |
| **动作连贯性/鲁棒性** | ACG (ICRA 2026)、KAN We Flow |

### 关键论文

#### ReinFlow（NeurIPS 2025 / 2026.01）

```
FM 策略 + 在线 RL 微调
  → 在 Rectified Flow / Shortcut Models 上做在线 RL
  → 注入可学习噪声作探索
  → 操作任务 +40% 成功率
  → 比 Diffusion RL 省 23-83% wall time

对你的意义：
  → FM 的后训练不只可以是 BC，也可以是 RL
  → 你的后训练方案里应该加入 RL fine-tuning 这个选项
```

#### Flow to One Step（2026.03）

```
从 CFM 教师 → 单步学生蒸馏
  → 用 IMLE (Implicit Maximum Likelihood Estimation) 保留多模态动作分布
  → 支持实时滚动时域重规划

对你的意义：
  → 你的小脑回路设计的核心使能技术
  → FM 蒸馏到 1 步 → 可以在 1000Hz 运行
```

#### FlowRAM（CVPR 2025）

```
区域感知 3D + Mamba + CFM
  → <4 步推理
  → RLBench SOTA，高精度任务 +12%

对你的意义：
  → 少步推理的实现参考
  → Mamba (State Space Model) 作为 backbone 的替代方案
```

#### Streaming Flow Policy（CoRL 2025）

```
动作轨迹 = 流轨迹
  → 在 FM 采样过程中流式输出动作
  → 更紧的感觉-运动闭环

对你的意义：
  → 小脑回路的流式动作输出方案
  → 不需要等完整的 ODE 解完再执行
```

#### ManiFlow（CoRL 2025）

```
Consistency Flow Training → 通用操作策略
  → 强泛化能力
  → 解决复杂双臂灵巧操作

对你的意义：
  → 灵巧手操作的 FM 策略参考
```

#### ACG: Action Coherence Guidance（ICRA 2026）

```
无需训练，测试时引导
  → 抑制人类 demo 中的 jerk/jitter
  → 在 RoboCasa、DexMimicGen 和真实 SO-101 上验证

对你的意义：
  → 你的遥操作数据里的噪声可以用 ACG 抑制
  → 测试时引导——不需要重新训练
```

#### Generative Predictive Control（2025）

```
采样 MPC + FM 的桥接
  → 对容易仿真但难演示的动态任务
  → 支持运行时热启动

对你的意义：
  → FM + MPC → 你的大脑(规划) + 小脑(执行) 的理论基础
```

## 三、力触觉 + 灵巧操作（2025-2026）

### 核心趋势

| 趋势 | 代表工作 |
|------|----------|
| **力作为一等信号** | DexForce、Adaptive Visuo-Tactile Fusion、3D Tactile ICRA |
| **跨传感器触觉迁移** | GenForce (Nature Comms 2026) |
| **零样本 Sim-to-Real** | Zero-Shot Dexterous Force-Based (Jan 2026) |
| **动觉示教采集触觉** | KineDex、DexTac、DexForce |
| **潜在触觉预测** | HTD (CMU)、Adaptive Visuo-Tactile Fusion |

### 关键论文

#### GenForce（Nature Communications，2026）

```
跨传感器触觉力迁移
  → 在一种触觉传感器上训练的力预测模型 → 迁移到其他类型传感器
  → 法向力误差降低 80%
  → 补偿材料硬度和老化效应

对你的意义：
  → 你的多传感器融合方案——不同硬件用不同触觉传感器的数据可以互通
  → 预训练触觉表征 + 跨硬件迁移
```

#### DexTac（2026.01）

```
手把手示教 → 多维度触觉数据 → 灵巧手策略
  → 捕获接触力分布 + 空间接触区域
  → 注射任务 91.67% 成功率
  → 比纯力控 baseline 高 31.67%

对你的意义：
  → 触觉数据的采集方法——动觉示教比遥操作快 2×
  → 触觉+力联合比纯力好 31.67%——数据支撑你的多模态设计
```

#### Zero-Shot Sim-to-Real Dexterous Force（2026.01）

```
完全在仿真中训练 → 零样本部署到真机灵巧手
  → 快速触觉仿真（密集虚拟触觉单元）
  → 电流→力矩标定（省去力矩传感器）
  → 首次实现多指手可控抓取 + 手中重定向，零样本

对你的意义：
  → 灵巧手数据的最大瓶颈（真机采集贵）→ 仿真可以解决
  → Sim-to-Real 的力触觉迁移方法
```

#### DexForce（IEEE RAL，2025）

```
从动觉示教中提取力信息 → 力知情的动作策略
  → 6 个接触丰富任务平均 76% 成功率
  → 没有力信息的策略 → 几乎为零

对你的意义：
  → 力信号的价值有量化证据——没有力 = 零成功率
  → 动觉示教采集力数据的 pipeline
```

#### Adaptive Visuo-Tactile Fusion（2025.05）

```
力引导的注意力融合
  → 自适应调整视觉 vs 触觉特征权重
  → 自监督未来力预测
  → 3 个接触丰富任务 93% 平均成功率

对你的意义：
  → 视觉和触觉的权重不需要手动设——网络自己学会"接触前看视觉，接触后信触觉"
  → 这就是你的 MoT Router 应该学到的行为
```

#### 3D Tactile Canonical + Force Pretraining（ICRA 2025）

```
3D 触觉的标准表征 + 力基础自监督预训练
  → 捕获局部/净力特征
  → 多指灵巧手，4 个精细任务 78% 平均成功率

对你的意义：
  → 触觉的预训练方法——自监督、力基础
  → 标准表征 → 跨传感器的泛化
```

#### KineDex（CoRL 2025）

```
动觉示教 + 触觉增强输入 + 力控部署
  → 74.4% 平均成功率
  → 比无力控版本高 57.7%
  → 数据采集比遥操作快 2×

对你的意义：
  → 力控的价值再次被量化：+57.7%
  → 数据采集效率是实用化的关键
```

#### HTD: Humanoid Transformer with Touch Dreaming（CMU，2025）

```
潜在空间预测触觉 → 人形机器人全身操作
  → 分离下肢平衡和上肢灵巧
  → 比 ACT baseline 高 90.9%（相对提升）
  → 5 个真实任务（插入、书籍整理、毛巾折叠、舀取、端茶）

对你的意义：
  → "Touch Dreaming"——在 latent 空间预测触觉，不需要真实触觉传感器就能训练
  → 你的预训练可能不需要大量真实触觉数据
```

## 四、预训练 + 后训练范式（2025-2026）

### 核心范式

```
共识的两阶段范式：

阶段 1 — 预训练（基础模型）：
  大规模、多具身、多模态数据
  → 学"世界是怎么运作的，机器人是怎么动的"

阶段 2 — 后训练（任务适配）：
  少量高质量任务数据
  → 适配到特定硬件、特定任务
  → 核心挑战：不忘记预训练的能力（灾难性遗忘）
```

### 关键方法

#### Parameter Merging（ICLR 2026）

```
方法：微调后模型的权重 和 预训练模型的权重 做插值
  → 一个模型同时继承通用能力 + 学会新任务
  → 在新任务的 OOD 变体上超过预训练和微调模型
  → 性能随预训练数据量增长
  → 支持终身/持续学习

对你的意义：
  → 你的后训练灾难性遗忘的解决方案
  → 比 LoRA 更激进（直接操作权重），效果可能更好
```

#### FLaRe（ICRA 2025）

```
大规模 RL 微调
  → 用稀疏奖励即可（不需要密集 reward engineering）
  → 79.5% 平均成功率（长序列移动操作，未见过环境）
  → +23.6% 仿真，+30.7% 真机
  → <1 天微调即可适配新具身
  → 泛化到训练数据之外的 tasks + embodiments

对你的意义：
  → RL fine-tuning 的标杆数字
  → "不到1天适配新具身"——你的硬件适配阶段可以参考
```

## 五、对你设计的影响

### 架构层面

```
最新趋势 → 你的设计调整：

1. Action Manifold 假说 (ABot-M0)
   → 你的 SE(3) 流形 FM 和这个一脉相承
   → 可以明确声称你的动作空间是 SE(3) × R^6 的乘积流形

2. 3D-aware VLM (SPEAR-1)
   → 视觉 backbone 应该有 3D 空间理解能力
   → 不只是 2D ViT

3. DiT backbone (ABot-M0, ABot-PhysWorld)
   → FM 的 backbone 可以用 DiT 而非简单的 MLP
   → Flow Matching + DiT = 更 expressive

4. Mamba/SSM (FlowRAM)
   → Transformer 不是唯一的 backbone 选择
   → 长序列（高频力触觉数据）可能更适用 SSM
```

### 训练层面

```
1. 预训练数据配方：
   → 跨具身 > 单具身（MiMo-Embodied 的驾驶→操作正迁移）
   → 3D 标注 > 纯 2D（SPEAR-1 的数据效率）
   → 力触觉标注价值极高（DexForce: 没有力 ≈ 零成功率）

2. 后训练方法：
   → Parameter Merging 可能是灾难性遗忘的最佳解
   → RL fine-tuning 在 BC 预训练之上有用（ReinFlow +40%）
   → 交互修正（FlowCorrect: 80% 修正已失败的 case）

3. 力触觉的预训练：
   → 跨传感器迁移可行（GenForce: -80% 误差）
   → 自监督、力基础的预训练（3D Tactile ICRA 2025）
   → Touch Dreaming: 不需要真实触觉也能训练触觉预测
```

### 工程部署层面

```
1. FM 蒸馏到 1 步 → 实时控制（Flow to One Step）
2. 流式动作输出 → 更紧的闭环（Streaming Flow Policy）
3. FM + MPC → 大脑规划+小脑执行（Generative Predictive Control）
4. ACG → 抑制 demo 噪声，测试时即可用
```
