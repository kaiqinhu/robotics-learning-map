> **状态：** 📝 待学习

## 背景：预训练只是开始

你花了几周训好一个通用策略——它在 LIBERO 上跑得不错。然后你把它部署到一台真机上，让它开一扇它从没见过的柜门。

它失败了。

不是因为模型不够好，而是因为**预训练和真机部署之间有一条鸿沟**：硬件不同（关节柔顺性、传感器噪声）、任务分布不同（你的柜门不在训练集里）、数据量极少（只有 20 条真机 demo）。而且更糟的是——你微调它学会开这扇门之后，它忘记了怎么开之前会开的那些门。

这就是后训练要解决的问题：**怎么用极少的新数据适配新任务，同时不丢掉预训练学到的通用能力？**

四种策略从简单到复杂：Parameter Merging（一行代码）、RL Fine-tuning（ReinFlow 做到 +40%）、LoRA/增量 MoE（只训练 1% 参数）、FlowCorrect（部署时不训练直接修）。

## 学完这一页你能

- 说出四种后训练策略的做法、适用场景、和计算成本
- 解释 Parameter Merging 为什么能防止灾难性遗忘（权重空间的插值直觉）
- 理解为什么 FM 比 Diffusion 更适合 RL 微调（ODE 采样更快 → 交互-更新循环更快）
- 设计力控风格的微调方案：不只输出位姿，还输出 (K(t), B(t), F_d(t))

## 核心驱动问题

1. **Parameter Merging 一行代码 `θ_merged = (1-α)·θ_finetuned + α·θ_pretrained`——为什么这么简单的东西能发 ICLR 2026？** 它在什么情况下会失效？
2. **ReinFlow 的"注入可学习噪声"为什么是关键？** FM 的 ODE 天然确定性 → 策略梯度没法算 → 怎么把 ODE 变 SDE？
3. **LoRA 和增量 MoE 各有什么 trade-off？** 多任务切换时加载不同 LoRA 权重 vs 让 MoE Router 自动判断——哪个更适合你的场景？
4. **力控风格微调：不同任务需要不同的阻抗参数（K, B, F_d）——FM 怎么同时输出位姿和力控参数？** 是分两个 head 还是同一个 head？
5. **从 50 条真机数据到泛化，最有效的策略是什么？** 数据增强？力控先验？还是直接 Parameter Merging？

## 1. 后训练的四种策略

### A. Parameter Merging（ICLR 2026）

```
方法：
  θ_merged = (1-α)·θ_finetuned + α·θ_pretrained
  
效果：
  → 同时保留通用能力 + 新任务能力
  → 在 OOD 变体上超过微调模型和预训练模型
  → 支持持续学习——加新任务不需要重新训练全部

优势：极其简单（一行代码），效果显著
```

### B. RL Fine-tuning

```
ReinFlow (NeurIPS 2025)：
  BC 预训练的 FM 策略 → 在线 RL 微调
  → 注入可学习噪声作探索（ODE→SDE，使策略梯度可计算）
  → 操作任务 +40% 成功率
  → vs Diffusion RL (DPPO)：操作任务省 23% 训练时间、腿式运动省 83% wall time

FLaRe (ICRA 2025)：
⚠️ 注意：这是 FLaRe (Allen AI, ICRA 2025)，不是 FLARE (NVIDIA, CoRL 2025)
  → 79.5% 长序列操作成功率
  → <1 天微调适配新具身
  → 只需稀疏奖励

关键：FM 比 Diffusion 更适合 RL 微调
      因为 ODE 采样更快 → RL 的交互-更新循环更快
```

### C. LoRA + 增量 MoE

```
LoRA：
  W' = W + A·B,  其中 A∈R^{d×r}, B∈R^{r×d}, r << d
  → 只训练 1% 参数
  → 多任务可切换（加载不同的 LoRA 权重）

增量 MoE：
  新任务 → 新 expert
  旧 expert 冻结
  Router 学习在任务间切换

  参考：MoT 的 Router 可以判断"这是开门的 token 还是拧瓶盖的 token"
```

### D. 交互修正（FlowCorrect, 2026）

```
不需要重新训练：
  部署时 → 人类通过 VR 给稀疏修正信号
  → 模型实时调整动作
  → 80% 修复已失败的 case

适用：那些"差一点就成功了但不想花时间重新训练"的场景
```

## 2. 力控风格微调

```
不同任务 → 不同力控风格：

  擦桌子：K 小（软、顺应），恒力 5N
  插销：  K 先大后小（先硬后软），力阈值触发
  抓鸡蛋：K 极小（极软），力上限 2N
  抓金属：K 大（硬），力上限 50N

做法：FM 不只输出位姿 → 还输出 (K(t), B(t), F_d(t))
     微调时只微调力控风格分支
```

## 3. 从开一扇门到开所有门——泛化

```
门的变异维度：尺寸、把手形状、推/拉、铰链位置

泛化策略：
  → Object-centric 表征：门 = 铰链 + 面板 + 把手（几何关系）
  → 力控泛化：门的力曲线形状本质相似
    （克服静摩擦 → 大力 → 动摩擦 → 小力）
  → 视觉泛化比力控泛化难得多
    → 你的策略应该"更信力触觉，更不信视觉"

关键挑战：从 50 条真机数据到泛化 → 数据增强 + 力控先验
```

---

## 4. 持续学习

```
灾难性遗忘：任务 B 微调 → 任务 A 成功率归零

解决方案优先级：
  1. Parameter Merging（最简单，ICLR 2026 验证有效）
  2. 增量 MoE（新任务=新 expert，冻结旧 expert）
  3. Replay（保留 5% 旧任务数据混合训练）
  4. EWC（惩罚重要权重的变化）
```
