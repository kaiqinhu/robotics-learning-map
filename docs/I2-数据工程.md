# 🟢 I2-数据工程

> **核心问题：** UMI（大量低精度）、ego（机器人视角）、dex（灵巧手）、真机实采（少量高精度）——四种数据怎么融合？SLAM 位姿和运动学位姿噪声性质不同，怎么处理？
>
> **预计时间：** 1 周
>
> **状态：** 📝 待学习

---

## 1. 四种数据源

| | UMI | ego | dex | 真机 |
|------|------|------|------|------|
| **位姿精度** | 低（SLAM 漂移 cm~dm 级） | 中 | 中 | 高（运动学 mm 级） |
| **力信号** | 通常无 | 可能有 | 可能有 | ✓ |
| **触觉** | 无 | 无/有 | 有 | ✓ |
| **量级** | 10K+ 轨迹 | 1K+ | 100+ | 10-100 |
| **能学什么** | 大概怎么动 | 机器人视角下怎么做 | 手指配合 | 精确怎么做 |

## 2. 分阶段训练

```
阶段 1 — UMI 预训练（大量，位姿噪声大）：
  → 学操作的整体语义（抓这→提→移→放）
  → 训练时加位姿噪声（模拟 SLAM 误差）
  → 模型学会"位姿不用太精确，方向对就行"

阶段 2 — ego + dex 预训练：
  → 增加机器人视角和灵巧手能力

阶段 3 — 真机微调（少量，高精度）：
  → 精细调整力控精度和触觉反应
```

---

## 3. 质量感知训练

```
每条数据加 metadata：
  {source: "umi", quality: 0.3, has_force: false, has_tactile: false}
  {source: "real", quality: 0.95, has_force: true, has_tactile: true}

模型输入：observation + quality_embedding
损失加权：低质量数据 → 损失权重低（不用精确匹配有噪声的 label）

缺失模态：
  UMI 缺力、缺触觉 → 用 [MISSING] learnable embedding 代替
  训练技巧：随机 drop 真机中的力/触觉 → 模型学会优雅退化
```

## 4. SLAM 位姿 vs 运动学位姿

```
SLAM 位姿误差：
  → 慢变漂移（不是白噪声）
  → 平移 cm~dm 级

运动学位姿误差：
  → 关节编码器精度 + 标定误差
  → 平移 mm 级

两者不能简单当成"高噪声 vs 低噪声"
  → 误差分布结构不同
  → 建模：T_umi = T_true · exp(ε_umi),  ε_umi ~ SLAM error distribution
```
