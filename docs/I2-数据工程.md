## 背景：数据决定了模型的上限

你的模型架构再精巧，如果数据喂错了，一切白费。具身模型的数据不是"多就是好"——**四种数据源有四种不同的噪声结构、精度等级、和缺失模态。**

```
UMI 手持设备采集   → 量大（10K+ 轨迹），但 SLAM 位姿漂移 cm~dm 级，没力触觉
ego 机器人视角     → 量大但单一视角，力触觉可能缺失
dex 灵巧手遥操作   → 手指动作精细，但数据量小（100+ 轨迹）
真机实采          → 黄金标准（力触觉齐全、位姿 mm 级），但只有 10-100 条
```

把它们一锅炖？SLAM 的 cm 级漂移和运动学的 mm 级精度不是"噪声大小不同"——是**噪声结构完全不同。** SLAM 漂移是慢变、空间相关的；运动学误差是高频、关节独立的。把它们当同一种噪声来处理，模型会学到错误的东西。

数据工程的核心问题：**怎么设计分阶段训练，让模型先学到"大致怎么做"（从大量低质量数据），再学到"精确怎么做"（从少量高质量数据）？**

## 学完这一页你能

- 列出四种数据源的特征矩阵（精度、模态齐全度、量级、成本）
- 设计三阶段训练方案：UMI 预训练 → ego+dex 增强 → 真机微调
- 解释为什么 SLAM 位姿和运动学位姿不能当成"高噪声 vs 低噪声"来简单处理
- 实现质量感知训练：低质量数据用 learnable quality embedding + 损失加权
- 处理缺失模态：用 [MISSING] token + 随机 drop 训练

## 核心驱动问题

1. **UMI 数据的位姿标签有 cm 级漂移——用这种数据预训练，模型会不会学到"位姿不重要"的错误先验？** 怎么在训练时注入噪声让模型学会"方向对就行，精度别信标签"？
2. **SLAM 漂移是慢变、空间相关的；运动学误差是高频、关节独立的——这两种误差结构对 FM 的向量场学习分别有什么影响？**
3. **分阶段训练时，各阶段的损失权重怎么设？** UMI 阶段力觉缺失——BC loss 的权重应该比真机阶段低还是高？
4. **质量感知训练："数据质量"怎么量化？** source 标签（UMI=0.3, real=0.95）够吗？还是需要更精细的 per-trajectory 质量估计？
5. **如果只有 20 条真机 demo，但想要力控泛化——数据增强和 sim-to-real 哪个优先？**

## 1. 四种数据源

| | UMI | ego | dex | 真机 |
|------|------|------|------|------|
| **位姿精度** | 低（SLAM 漂移 cm~dm 级） | 中 | 中 | 高（运动学 mm 级） |
| **力信号** | 通常无 | 可能有 | 可能有 | ✓ |
| **触觉** | 无 | 无/有 | 有 | ✓ |
| **量级** | 10K+ 轨迹 | 1K+ | 100+ | 10-100 |
| **能学什么** | 大概怎么动 | 机器人视角下怎么做 | 手指配合 | 精确怎么做 |

## 2. 分阶段训练

```
阶段 1 — UMI 预训练（大量，位姿噪声大）：
  → 学操作的整体语义（抓这→提→移→放）
  → 训练时加位姿噪声（模拟 SLAM 误差）
  → 模型学会"位姿不用太精确，方向对就行"

阶段 2 — ego + dex 预训练：
  → 增加机器人视角和灵巧手能力

阶段 3 — 真机微调（少量，高精度）：
  → 精细调整力控精度和触觉反应
```

---

## 3. 质量感知训练

```
每条数据加 metadata：
  {source: "umi", quality: 0.3, has_force: false, has_tactile: false}
  {source: "real", quality: 0.95, has_force: true, has_tactile: true}

模型输入：observation + quality_embedding
损失加权：低质量数据 → 损失权重低（不用精确匹配有噪声的 label）

缺失模态：
  UMI 缺力、缺触觉 → 用 [MISSING] learnable embedding 代替
  训练技巧：随机 drop 真机中的力/触觉 → 模型学会优雅退化
```

---

## 4. SLAM 位姿 vs 运动学位姿

```
SLAM 位姿误差：
  → 慢变漂移（不是白噪声）
  → 平移 cm~dm 级

运动学位姿误差：
  → 关节编码器精度 + 标定误差
  → 平移 mm 级

两者不能简单当成"高噪声 vs 低噪声"
  → 误差分布结构不同
  → 建模：T_umi = T_true · exp(ε_umi),  ε_umi ~ SLAM error distribution
```
