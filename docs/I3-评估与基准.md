## 背景：成功率 ≠ 模型好

具身模型评测有一个公开的问题：**大部分 benchmark 只测"位置到了没"，不测"力用对了没"。**

LIBERO 上的 99% 成功率听起来很美——但那是纯位置控制任务。一旦涉及接触操作（插销、抓鸡蛋、拧瓶盖），位置对≠力对。而且更麻烦的是：LIBERO 正在饱和（top 模型 >99%），区分度越来越低。RoboCasa 还只有 ~70%，RoboChallenge 真机评估只有 ~50%——这些才是真正能看出差距的战场。

但对你来说，最大的问题是：**力触觉操作没有标准 benchmark。** 你得自己定义评估指标、自己设计消融实验、自己说服别人你的模型在力控上比别人好。

## 学完这一页你能

- 列出主流 benchmark 的最新排行和各自的天花板
- 解释为什么 LIBERO 在饱和而 RoboCasa/RoboChallenge 还是硬仗
- 为力触觉操作设计一套评估指标（力跟踪误差、过度用力率、滑移率…）
- 设计标准的消融矩阵：模型组件 × 数据策略 × 蒸馏步数
- 说服别人：你的模型不仅在"位置到了没"上不差，在"力用对了没"上更好

## 核心驱动问题

1. **为什么 LIBERO 上 99% 的模型在真机上可能只有 50%？** 仿真-真机差距在哪里？力触觉在这个差距中扮演什么角色？
2. **怎么证明你的力触觉模块真的有用？** 消融实验的 gold standard 是什么——是去掉力触觉的成功率下降，还是力控指标本身的恶化？
3. **如果现有 benchmark 没有力控指标，你能不能改造它们？** 比如在 LIBERO 里加力传感器模拟，或者用 RoboChallenge 的真机平台做力控专项评估？
4. **消融实验中最容易被忽略的维度是什么？** 不是模型组件，而是数据——真机数据量 10→25→50→100 的 scaling 曲线才是你的工作最有说服力的证据。
5. **DobotWAM 的 Joint Dynamic Geometry Loss 是一个"物理约束进 loss"的方向——你能不能设计一个 Force-Tactile Consistency Loss 作为你的专有评估维度？**

越疆科技 2026.05.29 发布，LIBERO 99.25% 登顶。目前无公开论文/代码，但技术方向明确：

```
WAM > VLA：不只是"模仿动作"，而是"理解动作为什么这样做"

四大技术支柱（对你设计的启示）：
  ① 3D-Aware Spatial Representation  → 与 SPEAR-1 同方向，3D 是数据效率关键

→ ②④ 可以作为你的差异化：力触觉→物理约束 loss + 力控数据飞轮
```

### 对你的意义

```
力触觉操作目前在 benchmark 中几乎完全缺失：
  → 大部分 benchmark 只评估"位置到了没"
  → 力控质量、接触稳定性没有标准 benchmark

你需要：
  → 用 LIBERO/RoboCasa 验证位姿能力
  → 对力触觉，自己定义评估指标（见下节）
  → 或改造现有 benchmark（如 LIBERO 加力传感器模拟）
```

## 2. 力触觉评估指标

### 位置指标（常用但不够）

```
成功率：完成了多少任务
位姿误差：最终位姿离目标位姿的测地线距离
轨迹平滑度：轨迹的 jerk（加加速度）

→ 这些只能告诉你"位置对不对"
→ 不能告诉你"力用得对不对"
```

### 力控指标（你需要补充的）

```
力跟踪误差：||F_actual - F_target|| / ||F_target||
接触稳定性：接触状态的切换次数（"去掉接触→重新接触"的频次）
过度用力率：F_actual > F_safe_threshold 的时间占比
滑移率：物体滑移事件数 / 操作时长
能量效率：总注入能量 / 任务完成所需最小能量估计
响应时间：接触力从 0 上升到目标值的时间
```

## 3. 消融实验设计

### 标准消融维度

```
模型组件消融：
  - FM on SE(3) vs FM on R^6（验证 SE(3) 的必要性）
  - 有力触觉 vs 无力触觉（验证力信号的贡献）
  - MoE vs 单一 FFN（验证 MoE 的贡献）
  - 有 Reflow vs 无 Reflow（验证路径拉直的效果）
  - 有力控风格微调 vs 无（验证力控微调的效果）

数据消融：
  - UMI only vs UMI+真机（验证多源数据的价值）
  - 数据量 100% → 50% → 10%（验证数据效率）
  - 真机数据量 10 → 25 → 50 → 100（验证 scaling）

蒸馏消融：
  - 50步 vs 10步 vs 5步 vs 1步（验证速度-质量 trade-off）
```
