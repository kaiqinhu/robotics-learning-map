## 背景：为什么先看全景？

机器人动作生成不是只有一种做法。2023-2026 年间，四条技术路线并行演进——自回归离散、扩散模型、流匹配、掩码生成——每条路线背后是不同的数学原理、推理效率、和适用场景。

如果你一上来就钻进 Flow Matching 的公式里，你会错过一个关键事实：**FM 的优势（快、连续、可蒸馏）只有在和其他范式对比时才真正成立。** 面试官不会只问"FM 是什么"，他们会问"为什么选 FM 不选 Diffusion？"

本页不做深度推导——只在做深 FM 之前，先建立一张清晰的全景地图。

## 学完这一页你能

- 画出四种动作生成范式的对比表，说清各自的数学本质
- 解释为什么 Diffusion 在极难任务上还略占优，而 FM 在速度上占优
- 给出选择 FM 的三个充分理由，并对可能的质疑有回应
- 理解 "Much Ado About Noising" 争议的核心论点：生成式策略的优势到底来自哪里？

## 核心驱动问题

1. **这四种范式的数学本质区别是什么？**（不只是"做法不同"——是训练目标不同、采样方式不同、路径性质不同）
2. **为什么 Diffusion Policy 在 ToolHang 这种极难任务上还略优于 FM？** FM 的直线路径是优势还是局限？
3. **"Much Ado About Noising" 说生成式策略的优势被高估了——这个批评合理吗？** 如果优势来自迭代计算+随机性注入，而不是多模态建模，对你的 FM 路线意味着什么？
4. **MGP（掩码生成）声称比 Diffusion 快 35×——它有没有可能成为第三势力，取代 FM？**

---

> **核心问题：** 机器人策略有几种生成动作的方式？Flow Matching 只是其中一种——在做深 FM 之前，先看清全貌。

## 四大范式一览

```
1. 自回归离散 (Autoregressive Discrete VLA)
   RT-2, OpenVLA
   → 把动作切成 token，像说话一样逐个生成
   
2. Diffusion Policy
   Chi et al. 2023, DP3, DP4
   → 从噪声逐步去噪，生成连续动作
   
3. Flow Matching ← 你的主线
   π₀, ManiFlow, FlowRAM
   → 学向量场，ODE 运输概率

4. 掩码生成 (Masked Generative Policy)
   MGP (ICLR 2026), Discrete Diffusion VLA
   → mask 一部分 → 并行重建（不是逐个生成）
```

## 对比表

| | 自回归离散 | Diffusion | Flow Matching | 掩码生成 |
|---|---|---|---|---|
| **动作空间** | 离散 token | 连续 | 连续 | 离散或连续 |
| **生成方式** | 逐个 token 预测 | 逐步去噪 | ODE 积分 | 并行 mask 重建 |
| **推理步数** | = token 数 | 10-100 | 1-50 | <10 |
| **多模态建模** | ❌ 弱 | ✅ 强 | ✅ 强（路径直时弱） | ✅ 中 |
| **轨迹平滑** | ❌ 需要后处理 | ✅ 天然平滑 | ✅ 天然平滑 | ✅ 中 |
| **语言对齐** | ✅ 强（VLM 原生） | ⚠️ 需要额外设计 | ⚠️ 需额外设计（π₀ 做到了） | ⚠️ 需额外设计 |
| **代表模型** | RT-2(55B), OpenVLA(7B) | DP3, DP4, RDT-1B | π₀(3.3B), ManiFlow | MGP, Discrete Diff VLA |
| **真实速度** | 慢（token 逐个） | 慢（多步去噪） | 快（少步 ODE） | 快（并行） |

| FlowPolicy | FM | 0.85 | 0.82 | 强 |

**关键发现（2025）：** Diffusion 在极难任务（ToolHang）还略占优。FM 在速度上占优。两者在大多数任务上差距不大。

## 争议：生成式策略真的需要"生成"吗？

**"Much Ado About Noising"（2025.12）：**
```
评估了 Diffusion 和 FM 的多种变体后发现：
  → 优势可能不来自"多模态建模"
  → 优势来自"迭代计算 + 监督中间步骤"
  → 简单的两阶段回归策略 (MIP) 就能匹配 FM 的性能

这意味着：
  → 生成式框架的核心价值可能被高估了
  → 但 FM/Diffusion 仍然是 SOTA，没有更简单的方法全面超越
```

---

## 为什么不选别的，选 FM？

| 自回归离散 | 动作精度差（量化误差），不适合力控 |
| Diffusion | 太慢，不适合小脑回路的实时需求 |
| 掩码生成 | 太新，在力触觉操作上没有验证 |
| **Flow Matching** | **快 + 连续 + SE(3) 天然适配 + π₀ 已验证** |

## 2025 年共识：混合架构

```
VLM backbone（语义理解、任务规划）
    +
生成式 action head（连续动作生成）
    ├── FM（你的选择：快、可蒸馏、SE(3) 适配）
    ├── Diffusion（备选：极难任务可能更强）
    └── 自回归（如果你的核心优势是语言泛化）

→ π₀, GR00T, DiffusionVLA 都是这个架构
→ 关键 question：action head 选什么？
```
