# 🟢 I1-系统架构与部署

> **这是 FM 第三次出现。** C3 是 FM 在 R^d，M1 是 FM 在 SE(3) 上。这里，FM 作为一个策略部署到真实机器人——需要应对实时约束、通信延迟、和经典控制器的配合。
>
> **预计时间：** 2 周
>
> **状态：** 📝 待学习

---

## 背景：一个模型跑在 GPU 上 ≠ 一个机器人能工作

你的 FM 策略在离线测试里表现完美——50 步 ODE 生成平滑的 SE(3) 轨迹。然后你把它接到一台真机上：

- Transformer forward 花了 50ms → 最多 20Hz，但力控需要 1000Hz
- ODE 积分解完才输出第一个动作 → 机器人已经撞上去了
- 视觉帧 30Hz，力觉信号 1000Hz → 两种频率的信号怎么对齐？
- 网络抖了一下，FM 输出了一个物理上不可能的位姿 → 谁来做安全检查？

**模型 ≠ 系统。** 从 FM checkpoint 到一个能安全运行的机器人，中间隔着一整套工程架构：大脑(10Hz FM) + 小脑(1000Hz 阻抗控制) + 安全层(硬编码力上限) + 通信层(ROS2 topic) + 加速层(蒸馏/流式/跳过未来预测)。

## 学完这一页你能

- 画出大脑+小脑分层架构图，标注每层的频率、输入、输出
- 解释为什么端到端不可行（不是不想，是物理上做不到——Transformer 延迟 vs 力控频率）
- 比较四种 FM 加速方案：蒸馏到单步（IMLE）、流式输出（Streaming Flow Policy）、跳过未来预测（Fast-WAM）、FM+MPC 桥接
- 设计触觉反射的两条路径：进大脑（50ms，影响策略）vs 进小脑（2ms，直接修正力）
- 画出完整的 ROS2 topic 图：传感器 → 大脑 → 小脑 → 执行器

## 核心驱动问题

1. **为什么必须分层？** 把 FM 加速到 1000Hz 不就行了吗？——去算一下 Transformer forward + ODE 积分的最低延迟，看看物理上可不可能。
2. **Fast-WAM 说测试时不需要"想象未来"——这个洞察在你的系统架构里怎么落地？** 训练时加视频联合建模，推理时跳过生成——省下来的 600ms 用来做什么？
3. **触觉反射应该走哪条路径？** 滑移检测→夹持力+20%：2ms 小脑反射 vs 50ms 大脑处理——什么信号必须走快路、什么可以走慢路？
4. **Generative Predictive Control 说 FM+MPC 比纯 FM 更好——MPC 的局部优化纠正了 FM 的什么错误？** 这个桥接机制在你的力控场景里能用到吗？
5. **安全层怎么设计？** FM 输出附带不确定性——高不确定性时小脑自动变软(K↓)——这个逻辑怎么在 ROS2 里实现？

## 1. 大脑 + 小脑：为什么必须分层？

```
大脑（FM Transformer, ~10Hz）负责"想"：
  视觉+语言+触觉 → FM → SE(3) waypoints + 力目标

小脑（经典控制器, ~1000Hz）负责"做"：
  力传感器+触觉反射 → 阻抗控制器 → 关节力矩

为什么不能端到端？
  FM 的 Transformer forward 延迟 ~10-50ms → 最多 20-100Hz
  力控需要 500-1000Hz → 差一个数量级
  → 必须分层
```

## 2. 分层架构设计

```
┌────────────────── 大脑 (10Hz) ──────────────────┐
│                                                  │
│  Observation → FM Policy → SE(3) + Force targets │
│                    │                              │
│     FM 蒸馏到单步（Flow to One Step 2026）       │
│     或 Streaming Flow Policy (CoRL 2025)         │
│                    │                              │
└────────────────────┼──────────────────────────────┘
                     │  位姿目标 + 力目标
                     ▼
┌────────────── 小脑 (1000Hz) ─────────────────────┐
│                                                   │
│  F/T sensor → 重力补偿 → 外部力估计                │
│  Tactile → 滑移检测 → 触觉反射（硬编码或学习）      │
│                                                   │
│  τ = Jᵀ · [K·(x_d - x) + B·(ẋ_d - ẋ)] + g(q)    │
│       └────────── 阻抗跟踪 ──────────┘            │
│       + τ_tactile_reflex（触觉反射修正）           │
│                                                   │
└───────────────────────────────────────────────────┘
```

## 3. FM/WAM 部署加速

### Fast-WAM：测试时不需要"想象未来"（2026.03）

```
核心问题：WAM (World Action Model) 的价值来自哪里？
  → 训练时的视频联合建模？还是测试时的显式未来预测？

Fast-WAM 的回答（清华 MARS Lab + 星海图）：
  训练时：视频+动作联合训练 ✅（关键——改善世界表征）
  测试时：跳过未来帧生成，单次前向直接输出动作 ✅
  
  效果：LIBERO 97.6%, RoboTwin 91.8%
       推理延迟 190ms，比 imagine-then-execute 快 4×+

  消融：去掉训练时的视频联合建模
        → RoboTwin 91.8% → 83.8%
        → 真机毛巾折叠 ~80% → 10%
        → 视频建模的价值在训练时，不在测试时！

架构：Wan2.2-5B MoT（视频 DiT + 动作专家 DiT, 6B total）
```

→ 对你的启示：预训练阶段加视频联合建模改善表征，推理时跳过生成，直接输出动作。

### 蒸馏到单步

```
教师：CFM, 50 步 ODE → 高质量 SE(3) 轨迹
学生：单步预测，T_1 = f_θ(T_0, observation)

技术：IMLE (Implicit Maximum Likelihood Estimation)
      → 保留多模态分布（多种可能的抓取方式）
      → 而不是回归单一的"平均轨迹"

效果：1 步 → 1000Hz 可行
代价：质量略逊于 50 步，但对大多数场景够用
```

### 流式动作输出

```
Streaming Flow Policy (CoRL 2025)：
  不要等 ODE 完整解完再执行
  → FM 采样过程中边解边输出动作
  → 更紧的感觉-运动闭环
  → 参考：类似 MPC 的滚动时域，每个时间步只执行第一个动作
```

### FM + MPC 桥接

```
Generative Predictive Control (2025)：
  FM 作为 MPC 的"提案生成器"
  → FM 生成候选轨迹
  → MPC 在候选附近做局部优化
  → 结合 FM 的全局规划 + MPC 的局部精确
```

## 4. 触觉反射

```
两种路径：

路径 1 — 进入大脑（慢，50ms延迟，可学习）：
  Tactile image → CNN → embedding → FM Transformer
  → 影响全局策略（"这个东西很滑，用更大的力"）

路径 2 — 进入小脑（快，2ms延迟，硬编码或轻量 MLP）：
  Tactile image → 轻量 MLP → 反射信号
  → 直接修正力目标
  → 滑移检测 → 夹持力 +20%（立即执行，不等大脑）

设计原则：
  → 稳定相关的放小脑（力上限、滑移检测）
  → 策略相关的放大脑（抓取方式、任务规划）
```

## 5. ROS2 与实时通信

```
大脑-小脑通信：
  → ROS2 topic: /fm_policy/target {T_desired, F_desired, K, B} @ 10Hz
  → 小脑订阅 → 插值到 1000Hz（SE(3) 上的样条插值）

传感器 → 大脑：
  → 视觉: ROS2 topic /camera/rgbd @ 30Hz
  → 力觉: ROS2 topic /ft_sensor/wrench @ 1000Hz（降采样后 100Hz 给大脑）
  → 触觉: ROS2 topic /tactile/image @ 60Hz

安全层：
  → 小脑监测力上限 → 超限则紧急停止（硬编码，不经过大脑）
  → FM 策略输出附带不确定性 → 高不确定性时小脑降低 K（变软）
```
