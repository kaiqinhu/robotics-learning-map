## 背景：机器人没有 ImageNet

LLM 有互联网全量文本，视觉模型有 ImageNet/LAION。机器人呢？**每条操作轨迹都要花真金白银去采集，一条 demo 可能值几百块。**

这就是具身模型的根本瓶颈：「数据不够」。解决方法不是等数据变多——而是设计预训练目标，把有限数据的每一滴价值榨干。

2025 年的共识是：**单一的 BC loss 不够。** 你需要多目标联合预训练：
```
BC (Behavior Cloning)     → 学会"怎么做"
MAE (Masked Autoencoding) → 学会"看到什么"（不需要标签！）
Contrastive               → 学会"视觉和触觉是一回事"
World Model               → 学会"下一步会发生什么"
Force Prediction           → 学会"用了多大力"
```

但每个目标的成本不同——MAE 不需要标签，World Model 需要序列数据，Force Prediction 需要 F/T 传感器。**你怎么排布这五个目标？哪个先、哪个后、各自的 λ 怎么设？**

## 学完这一页你能

- 解释为什么 SPEAR-1 只用 20× 更少的机器人数据就能匹敌 π₀（3D 空间理解 = 数据效率）

- 说清 ABot-PhysWorld 的 DPO 后训练为什么能"抑制非物理行为"

## 1. 预训练目标的层级

```
L1 行为克隆 (BC)：                有动作标签的数据
  FM 匹配演示动作                  需要：遥操作/demo 数据

L2 掩码自编码 (MAE)：              所有数据（有标签无标签都行）
  随机 mask → 重建                 需要：任何观测数据

L3 对比学习 (Contrastive)：         配对数据
  视觉-触觉对齐、力-视觉对齐       需要：多模态配对

L4 世界模型 (World Model)：         序列数据
  预测下一时刻的 latent state      需要：交互序列

L5 力预测自监督：                   力触觉数据
  预测接触力、滑移                 需要：F/T 传感器标签

预训练损失：
  L = L_BC + λ₁·L_MAE + λ₂·L_contrastive + λ₃·L_world + λ₄·L_force_pred
```

---

## 2. 各目标详解

### 行为克隆 (BC)

```
最简单、最直接的预训练：
  输入：observation
  输出：action trajectory
  损失：FM 的 CFM loss

问题：只能复现数据中的行为，缺乏"创造力"
但：如果数据量够大、质量够高 → BC 本身就很强（参考 Octo, π₀）
```

### 掩码自编码 (MAE)

```
不需要标签，所有数据都能用：

触觉 MAE：mask 80% tactile image patches → 重建
力觉 MAE：mask 时间窗口 → 重建力序列
时空联合 MAE：mask 视觉+力+触觉的时空块 → 联合重建

→ 对数据量要求低（自监督）
→ encoder 学会"物理常识"
```

### 对比学习

```
视觉-触觉对比：
  "我看到这里有个棱角" ↔ "我摸到这里有个棱角"
  正对：同一物体同一位置的视觉+触觉
  负对：不同物体

力-视觉对比：
  "这东西看起来很大" ↔ "提起来果然很重"
  → 从视觉预测力的预期
```

### 世界模型

```
学一个预测"下一步发生什么"的模型：
  (当前 latent state, 动作) → 下一时刻的 latent state

要预测未来 → latent state 必须包含物理信息
→ 这个 latent state 就是策略的好输入

参考：ABot-PhysWorld (14B DiT, DPO 后训练抑制非物理行为)
```

## 3. 数据效率

### 预训练数据的量级参考

```
UMI 数据（低成本大量低精度）：    10K+ 条轨迹
ego 数据（机器人自主运行）：      1K+ 条
dex 数据（灵巧手遥操作）：        100+ 条
真机实采（最高质量）：            10-100 条

预训练策略：
  大量 UMI → 预训练"大致怎么动"（位姿精度低但动作语义对）
  真机数据 → 微调"精确怎么动"（位姿精度高、有力触觉）
```

### SPEAR-1 (CVPR 2026) 的数据效率启示

```
用 3D-aware VLM（SPEAR-VLM: PaliGemma + MoGe depth encoder）
在 Franka/WidowX 真机上匹敌 π₀-FAST 和 π₀.₅，但只用 20× 更少的机器人演示
关键：200K 非机器人 2D 图像 + 3D 标注（深度、bbox、距离）
  → 3D 空间理解 = 数据效率的关键
  → 你的视觉 encoder 应该是 3D-aware 的
```
